[ 장바구니 기능 분석 ]

1. 정의 : 구매의사가 있는 상품을 저장하는 기능
2. 데이터 : 상품 고유코드로 구분하여 저장함
3. 장바구니 순서 : 
    (1) 특정 상품에서 장바구니 버튼 클릭
    (2) 장바구니 담김 메세지 사용자에게 전달
    (3) 장바구니 리스트 버튼을 제공하여 클릭하면 장바구니를 보여준다
    (4) 장바구니 리스트에서 상품 및 토탈 합계를 제공
        - 상품제거기능
        - 같은 상품 개수 증감 기능
        - 구매 버튼 제공(상품 구매페이지로 이동)
4. 장바구니 로그인 계정연결
    (1) 로그인 없이 임시 저장소에 저장함
    (2) 로그인 했다면 로그인 계정의 DB에 저장함
    (3) 로그인 없이 저장한 장바구니를 구매할 경우
        로그인 연결 후 장바구니는 로그인 계정 DB에 저장함

5. 장바구니 구현 포인트
    (1) 장바구니 담기 버튼 클릭 시 : DB에 입력이 우선 임
    (2) 장바구니 읽을 때 : 가장 먼저 DB데이터 조회임!
        -> 임시 저장일 경우 로컬스토리지를 사용할 것임
    (3) 로컬스토리지 데이터 변환
        -> 읽어온 로컬스토리지는 객체로 변환
        -> 변환 된 객체에 변경사항을 반영
        -> 다시 로컬스토리지에 저장
